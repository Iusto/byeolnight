# 🌌 별 헤는 밤 (Byeolnight)

> **AI, 실시간 통신, 인증 자동화**를 통한 우주 코미니티 백어드 시스템
> 단순 기능 구현이 아니라, 운영 환경·보안·테스트·CI/CD까지 고도한 실무형 프로젝트

---

## 프로젝트 개요

* 시작: 2025.05 \~ : 2025.05 \~ \uc진행 중
* 개발 형태: 100% 개인 구조 및 구현 (백엔드 중심)
* 목표: 도메인 중심 구조에 AI 기능과 자동화 역할을 합칠해 가장 효율적인 테스트 가능 백어드 시스템 구성

---

## 그룹에서 선택한 기술의 필요성

현재 프로젝트에서 적용한 기술은 다음과 같은 것으로, 각 기술의 개선 필요성과 선택 이유를 Outven 프로젝트와 비교하여 명시합니다.

| 기술 단위    | Outven (기전)        | Byeolnight (개선)               | 선택 이유                   |
| -------- | ------------------ | ----------------------------- | ----------------------- |
| 인증/세션    | HttpSession 기반     | JWT + Redis TTL 기반            | 보안성 (다중 로그인 가능), 확장성 가지 |
| 패스 구조    | Controller 최종자 구조  | Enum 기반 분기 + 게시판 모듈화          | 모델화된 구조·유지보수성           |
| 배포       | WAR + Tomcat 직접 설정 | Docker + GitHub Actions CI/CD | 자동화 구조, 경험 자리 가지        |
| 환경/인터페이스 | JSP + Thymeleaf    | REST API + React (테스트 단계)     | API 우선 설계, 인증 연동 테스트 가능 |
| 실시간 구조   | 미구현                | WebSocket + STOMP             | 채팅/알림 등 실시간 구조 대응       |
| AI 연동    | 미구현                | Galaxy ML + Selenium          | 자동 게시/분류 기능, 확장성        |

---

## 기술 선택 가이드

* **Java 21 + Spring Boot 3.2**: 최신 LTS 버전 기반, Virtual Thread 적용성 확보 (고성능 IO 작업 대비)
* **JPA + QueryDSL**: 도메인 모델 중심 구조, 조건 검색의 타입 안정성 확보
* **Redis + JWT + Spring Security**: 세션 TTL 기반 인증, 무상태 구조, 다중 로그인 제어 가능
* **WebSocket + STOMP**: SSE와 비교하여 양방향 통신, 채팅/알림에 최적화
* **Swagger 3.0 + Postman**: 인증 포함 API 테스트 자동화, 문서화 통합
* **Docker + GitHub Actions**: 운영 환경과 동일한 로컬 구성, 수동 배포 제거
* **클린 아키텍처**: 각 계층 책임 명확화, 테스트 용이성 확보

## 개요 기능

| 영역    | 주요 내용                                                 |
| ----- | ----------------------------------------------------- |
| 인증/보안 | JWT + Redis TTL 기반 인증, BCrypt 암호화, Spring Security 적용 |
| 게시판   | 게시글/댓글 CRUD, Enum 기반 게시판 분기 처리, Soft Delete, 페이징      |
| 실시간   | WebSocket + STOMP 기반 채팅방 구조, 이벤트 비동기 처리               |
| 자동화   | 뉴스 크롤러 (Selenium), 은하 이미지 자동 분류 (Galaxy ML CNN)       |
| 배포    | GitHub Actions → Docker 이미지 빌드 → EC2 배포               |
| 문서화   | Swagger 문서 자동화, 인증 포함 테스트 가능                          |
| 테스트   | JUnit + Mockito 기반 단위/통합 테스트, 만료 토큰 등 예외 케이스 시나리오 테스트 |

## 패턴 구조

```bash
byeolnight/
├── domain/            # Entity, Repository (도메인 중심 구조)
├── application/       # Service 계층 (비즈니스 로직)
├── api/               # Controller 계층 (HTTP 진입점)
├── infrastructure/    # 외부 시스템 및 보안 연동 (Redis, Email 등)
├── dto/               # Request / Response DTO
├── util/              # 공통 유틸리티
├── test/              # 테스트 코드
```

## 해외 성적 및 회고

* **Outven 한계 극복**:

  * 구조적 복잡도, 세션 충돌, WAR 의존 배포 방식 등 실운영 문제 발견
  * Byeolnight에서 JWT 인증, Redis 세션 분리, Docker 기반 운영 환경으로 개선

* **실전 설계 감각 향상**:

  * 단순 기술 구현이 아닌, 인증/배포/보안/테스트를 통합한 실제 운영 가능한 백엔드 구조 체득

* **비즈니스 중심 사고 확립**:

  * 기술은 목적이 아닌 수단, 기능보다 구조, 구조보다 문제 해결에 초점
